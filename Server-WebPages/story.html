Our Story

Our journey into the world of server administration and web hosting began with a simple idea: to create a versatile, multi-site hosting solution from scratch. Little did we know that this adventure would lead us through a maze of technical challenges, learning experiences, and ultimately, to a fully functional server capable of hosting multiple websites.

The Beginning

It all started with the installation of Oracle VirtualBox. We needed a virtual environment to host our server without the overhead of a graphical interface.

Choosing the OS

After careful consideration, we settled on Ubuntu Server—a lightweight, open-source OS perfect for our needs.

Setting Up the VM

We created a new Ubuntu Server VM and proceeded with the installation. This was our first milestone—a clean slate with just a terminal interface.

Network Configuration

We configured static IP addressing to ensure our server would always be accessible at the same address—crucial for managing multiple sites.

Installing Nginx

With the base system ready, we installed Nginx, our chosen web server software, marking the beginning of our hosting capabilities.

Configuring Multiple Sites

We dove into Nginx configuration, learning how to set up multiple sites on a single server—a key feature of our project.

Overcoming Challenges

Throughout the process, we faced and overcame numerous challenges, from permission issues to network configurations.

The Final Product

After much effort and learning, we had a fully functional multi-site server, ready to host and serve multiple websites efficiently.

Challenges We Faced

Our journey was far from smooth sailing. We encountered numerous challenges that tested our problem-solving skills and deepened our understanding of server administration. Here are some of the major hurdles we overcame:

Our first major challenge was setting up the network on the virtual machine. We needed to ensure that the server was accessible via IP within the local network, but the default network setup in VirtualBox didn't allow for this.

Solution: We resolved this by configuring the VM's network adapter to use bridged networking. This allowed the VM to get an IP address directly from the router, making it accessible within the local network.

Even after configuring bridged networking, we faced issues when trying to access the server from other devices using the IP. The server's IP was changing every time it rebooted, causing inconsistent access.

Solution: We manually configured a static IP for the server by editing the network configuration files (/etc/netplan/*.yaml). This ensured that the server always used the same IP address.

After installing Nginx, we struggled with configuring it to host multiple websites. While the default Nginx page worked fine, setting up separate sites in their respective directories proved challenging.

Solution: We created different directories for each website and modified the Nginx configuration files to point to the correct root directories for each site. This required a deep dive into Nginx's configuration syntax and structure.

When trying to access our hosted sites, we were greeted with 403 Forbidden errors. This was due to incorrect file permissions preventing Nginx from accessing the files in our custom directories.

Solution: We used the chown and chmod commands to change file permissions and ownership, allowing Nginx's www-data user to access the files. This required learning about Linux file permissions and ownership concepts.

We encountered several errors in our Nginx configuration files, particularly when setting up the root directory or index files in the server blocks.

Solution: Through careful review and debugging of the configuration files, we corrected the errors. This process improved our understanding of Nginx's configuration structure and syntax.

Technologies Used

Our server setup journey involved a variety of technologies and tools. Each played a crucial role in bringing our multi-site hosting solution to life:

Ubuntu Server (VM)

A lightweight Linux server distribution used as the base operating system for our server. We chose Ubuntu Server for its stability, extensive documentation, and strong community support.

VirtualBox

>

A free and open-source virtualization software used to create and manage the Ubuntu Server virtual machine. VirtualBox allowed us to run our server in an isolated environment, making experimentation and configuration safer and more flexible.

Nginx

A high-performance web server used to host multiple websites. We configured Nginx to serve different sites on the same machine using virtual hosts, leveraging its efficiency and versatility.

Netplan (Networking)

A tool used to configure networking on Ubuntu, specifically to set up a static IP for consistent access. Netplan simplified the process of network configuration, allowing us to ensure our server was always reachable at the same address.

chmod & chown (File Permissions)

These Linux commands were crucial in adjusting file and folder permissions to allow Nginx to access and serve website content. Understanding and using these commands was key to resolving many of our initial setup challenges.

Current Server Features

After overcoming various challenges and fine-tuning our setup, our server now boasts several key features:

Nginx is installed and configured to serve multiple websites from the server. It listens on HTTP port 80 and responds with content from different directories based on the requested domain or path.

The server is configured to host multiple websites by setting up different root directories under the Nginx configuration. Each site can be accessed via its respective IP-based URL.

Our server currently serves static content, such as HTML files, with basic styling (CSS) for a clean user experience. This forms the foundation for more complex dynamic content in the future.

The server is accessible from other devices within the local network using its IP address. This allows for easy testing and development across multiple devices.

We've implemented a well-organized file structure, with separate directories for each hosted website. This makes management and updates easier and more intuitive.

The server is configured to allow SSH access, enabling remote management and configuration. This is crucial for maintaining the server without needing physical access.

What's Next?

While we've made significant progress, our journey is far from over. Here are some exciting improvements and features we're planning to implement:

Enable HTTPS

Our next major step is to set up SSL certificates to secure the websites and ensure HTTPS is used for all web traffic. We plan to explore Let's Encrypt for obtaining free SSL certificates.

Improve Website Management

We aim to create a more streamlined system for managing multiple websites with different configurations. This might involve developing custom scripts or exploring content management systems.

Implement Firewall and Security Hardening

To enhance security, we plan to set up and configure a firewall to restrict unnecessary incoming traffic and only allow specific ports. We'll also apply security patches and ensure that the server is hardened against potential vulnerabilities.

Deploy Web Applications and Databases

We're excited to explore deploying web applications and database services on the server. This will involve setting up environments for languages like PHP or Python, and database systems like MySQL or PostgreSQL.

Implement Backup Strategy

To protect our websites and databases from data loss, we plan to implement a robust backup strategy. This will likely involve automated backups to both local and cloud storage solutions.
